## BMS Master Skill

`/bms` is a master skill that activates the full bmsuisse coding context in one command. It routes to the relevant sub-skills based on a sub-command and always enables caveman communication style.

### Sub-commands

| Command       | Loads                                                                                               | Good for                                    |
| ------------- | --------------------------------------------------------------------------------------------------- | ------------------------------------------- |
| `/bms`        | coding-guidelines-sql ┬и coding-guidelines-python ┬и coding-guidelines-typescript ┬и fabricks-glossary | General session Рђћ all core standards active |
| `/bms sql`    | coding-guidelines-sql ┬и sql-optimization ┬и fabricks-glossary                                        | SQL authoring, reviews, refactoring         |
| `/bms python` | coding-guidelines-python                                                                            | Python development, PR reviews              |
| `/bms data`   | data-modeling-dimensional ┬и fabricks-glossary                                                       | Designing or reviewing fact/dim tables      |

For deep optimization work, chain with the specialized skills:

- SQL benchmarking Рєњ `/databricks-sql-autotuner`
- Python benchmarking Рєњ `/python-autotuner`

### How it works

```
/bms sql
  Рћѓ
  РћюРћђ reads bms/SKILL.md          (master router, ~2.5k chars)
  Рћѓ
  РћюРћђ runs load_skills.py sql     (helper script)
  Рћѓ     РћћРћђ concatenates:
  Рћѓ           coding-guidelines-sql/SKILL.md
  Рћѓ           sql-optimization/SKILL.md
  Рћѓ           fabricks-glossary/SKILL.md
  Рћѓ
  РћћРћђ applies all rules + caveman mode Рєњ "BMS SQL active."
```

The helper script at `skills/bms/scripts/load_skills.py` concatenates the relevant `SKILL.md` files and prints them as combined content. Each sub-skill remains the single source of truth Рђћ no duplication.

### Install

**Plugin (recommended)** Рђћ installs bms + all sub-skills as a self-contained plugin:

```bash
claude plugin marketplace add bmsuisse/skills
claude plugin install bms@bmsuisse-skills
```

**skills CLI** Рђћ installs the bms skill only (sub-skills must already be installed):

```bash
bun x skills add https://github.com/bmsuisse/skills --skill bms  # recommended
npx  skills add https://github.com/bmsuisse/skills --skill bms  # alternative
```

### Plugin path resolution

When installed as a plugin, `load_skills.py` resolves sub-skill paths via the `CLAUDE_PLUGIN_ROOT` environment variable that Claude Code injects at runtime. In the repo, it falls back to the repo-relative path. This means the script works correctly in both contexts without configuration.
