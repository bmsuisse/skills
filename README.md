## ūüöÄ Installation

The `skills` CLI reads directly from GitHub ‚ÄĒ no registration needed.

### Install a specific skill

```bash
npx skills add https://github.com/bmsuisse/skills --skill <skill-name>
```

### Install all skills

```bash
npx skills add https://github.com/bmsuisse/skills
```

The CLI auto-detects your agent platform and installs into the right directory:

| Platform | Directory |
|---|---|
| Claude Code | `.claude/skills/` |
| Cursor | `.cursor/rules/` |
| Codex CLI | `~/.codex/skills/` |
| Antigravity | `.agents/skills/` or `.agent/skills/` |
| Gemini CLI | auto-detected |

### ūüĒć Browse on skills.sh


> Skills appear on the [skills.sh leaderboard](https://skills.sh) automatically based on install telemetry ‚ÄĒ no manual registration required.

## ūüõ†ÔłŹ Adding a Skill

Use **skill-creator** to guide you through the full lifecycle ‚ÄĒ drafting, test cases, evaluation, and iteration:

```bash
npx skills add https://github.com/anthropics/skills --skill skill-creator
```

Then ask your agent: *"Help me create a skill for X"* and it will walk you through intent capture, writing the `SKILL.md`, running test cases, reviewing results, and iterating until the skill is ready.

### Structure

Each skill lives in its own directory under `skills/`:

```
skills/
‚ĒĒ‚ĒÄ‚ĒÄ your-skill-name/
    ‚Ēú‚ĒÄ‚ĒÄ SKILL.md          ‚Üź required: instructions + YAML frontmatter
    ‚Ēú‚ĒÄ‚ĒÄ scripts/          ‚Üź optional: helper scripts
    ‚Ēú‚ĒÄ‚ĒÄ references/       ‚Üź optional: reference documentation
    ‚ĒĒ‚ĒÄ‚ĒÄ assets/           ‚Üź optional: templates, icons, etc.
```

### SKILL.md format

```markdown
---
name: your-skill-name
description: What the skill does and when to trigger it. Be specific ‚ÄĒ this is how the agent decides whether to use it.
---

# Your Skill

Instructions for the agent to follow...
```

### Workflow

1. Create a branch: `git checkout -b skill/<skill-name>`
2. Create `skills/<skill-name>/SKILL.md`
3. Merge to `main` ‚ÄĒ CI auto-updates the **Available Skills** table above
